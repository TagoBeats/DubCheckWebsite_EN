§ 07 · Questions

The five questions a delivery manager will ask first.

If the answer isn't here, the docs at docs.dubcheck.io have the full schema, the measurement source and a changelog for every profile bump.
Q/01Which loudness meter sits under the hood?

A BS.1770-4 K-weighted integrator with –70 / –10 LU gating. We publish per-sample parity tests against the EBU compliance suite; identical numbers to the reference.

Q/02Do you keep my audio after the report?

No. Raw audio is purged 30 days after report issue — or 90 days on Studio Batch. The signed report, the hash and the spec profile version stay indefinitely.

Q/03What happens if the spec changes mid-delivery?

Every report is pinned to the profile version in effect on QC date. A profile bump doesn't retroactively invalidate a signed report — you still have the audit trail.

Q/04Can I run DubCheck from my DAW session?

Yes. dub-cli and a small AAX/AU helper can submit a bounce straight from Pro Tools or Nuendo, using the current session's clock and sample rate.

Q/05Is the signed PDF actually accepted by delivery ops at the big streamers?

It's designed to sit alongside — not replace — the platform's own ingest QC. What the report gives you is a deterministic, reproducible pre-check on your side, so the first upload passes instead of the third. Several post houses in Berlin and Hamburg use it as a gating step before handoff.
